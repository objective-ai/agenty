Add three more server actions to `src/lib/actions/auth.ts`:

**setupChildAccount:**
```typescript
export async function setupChildAccount(formData: FormData): Promise<{ success?: boolean; kidEmail?: string; error?: string }>
```
- Verify parent is authenticated: `const { data: { user: parent } } = await supabase.auth.getUser()` — return error if no parent
- Extract `pin` (string, must be exactly 6 digits) and `displayName` (string, default "Adventurer") from formData
- Validate PIN is exactly 6 numeric digits, return error if not
- Generate synthetic email: `player-${crypto.randomUUID()}@agenty.local`
- Use `supabaseAdmin.auth.admin.createUser()` with:
  - `email`: synthetic email
  - `password`: pin (Supabase handles hashing — do NOT hand-roll)
  - `email_confirm: true` (skip verification for synthetic account)
  - `user_metadata: { display_name: displayName, parent_id: parent.id }`
- Explicit profile upsert via `supabaseAdmin.from("profiles").upsert({ id: kidUser.user.id, display_name: displayName || "Adventurer" }, { onConflict: "id" })` — requirement AUTH-03 says not reliant on DB trigger alone
- Return `{ success: true, kidEmail }` — the kid email is needed for PIN login (stored client-side)

**loginWithPin:**
```typescript
export async function loginWithPin(kidEmail: string, pin: string): Promise<{ success?: boolean; error?: string; unlockAt?: string; remainingSeconds?: number; attemptsRemaining?: number }>
```
- Constants: `MAX_ATTEMPTS = 5`, `LOCKOUT_WINDOW_MINUTES = 15`
- Resolve the kid's profile_id: call `supabaseAdmin.auth.admin.getUserByEmail(kidEmail)` to get the user, then use `user.id` as profileId. If no user found, return `{ error: "wrong_pin" }` (don't reveal whether account exists).
- Check rate limit: query `supabaseAdmin.from("pin_attempts").select("attempted_at").eq("profile_id", profileId).gte("attempted_at", fifteenMinutesAgo).order("attempted_at", { ascending: false })`
  - Use `new Date(Date.now() - LOCKOUT_WINDOW_MINUTES * 60 * 1000).toISOString()` for the window filter
  - If `attempts.length >= MAX_ATTEMPTS`: calculate unlockAt from the oldest attempt in window + 15 min, return `{ error: "tactical_lockdown", unlockAt, remainingSeconds }`
- Attempt sign-in: `supabase.auth.signInWithPassword({ email: kidEmail, password: pin })`
- On failure: insert into pin_attempts `{ profile_id: profileId }`, return `{ error: "wrong_pin", attemptsRemaining: MAX_ATTEMPTS - attempts.length - 1 }`
- On success: delete all pin_attempts for this profileId (clear history), return `{ success: true }`
